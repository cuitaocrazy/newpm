## 五、数据恢复方法

### 5.1 数据库恢复

```bash
# 选择要恢复的备份文件（以 20260307 为例）
gunzip < /backup/newpm-mysql/newpm-20260307.sql.gz | \
  kubectl exec -i mysql-0 -n newpm -- \
  mysql -u root -ppassword --default-character-set=utf8mb4 ry-vue
```

**恢复时间预估：** < 1 分钟（数据量较小）

### 5.2 附件文件恢复

```bash
# 从本地 Mac 恢复附件到生产服务器
rsync -avz ~/backup/newpm-attachments/ \
  k3s001:/var/lib/rancher/k3s/storage/pvc-01f86b13-e8e1-4a3e-a152-1c1449fecf1b_newpm_upload-pvc/
```

## 七、备份脚本说明

### 服务器备份脚本 `/usr/local/bin/backup-newpm-db.sh`

- 进入 `mysql-0` Pod 执行 `mysqldump`
- 使用 `--single-transaction` 保证备份期间业务不受影响（热备份）
- 压缩后存储，节省磁盘空间
- 自动清理 30 天前的旧备份

### 本地同步脚本 `~/backup/sync-newpm-db.sh`

- 通过 SSH 密钥认证（`~/.ssh/id_ed25519`）连接服务器，无需密码
- 仅同步新增/变更文件，断点续传
- 自动清理本地 30 天前的旧备份
