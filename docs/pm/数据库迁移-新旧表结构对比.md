## 四、注意事项

1. **字典兼容性**：`sys_htzt`（合同状态）以旧库 8 种状态值为准，需向新库补充 `0`（谈判中）、`6`（已付完-结项）、`7`（已签-无需付款）三个值。
2. **字典补充**：`sys_fkzt`（付款状态）旧库有 9 种值，新库仅 7 种，需补充 `YKQK`、`FKWZ`、`BZHK` 三个值。
3. **迁移顺序**：建议顺序为 客户 → 联系人 → 区域 → 项目 → 审核 → 合同 → 款项 → 项目合同关联 → 附件 → 团队确认，避免外键依赖问题。
4. **AUTO_INCREMENT**：迁移完成后需根据实际最大 ID 重置各表的 `AUTO_INCREMENT` 值。
