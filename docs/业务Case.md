# 项目信息管理系统 — 业务 Case 文档

> 版本：1.1 | 最后更新：2026-03-31
> 描述各核心业务场景的完整操作流程与角色交互

---

## 一、业务背景

本系统服务于软件开发型企业，核心业务价值是：

1. **全生命周期管控**：从立项申请到项目结项，每个阶段都有可追踪的记录
2. **财务数据准确**：合同、款项、收入确认三层财务数据互相关联
3. **人天精细统计**：通过日报系统自动汇总每个项目/任务的实际工时
4. **多级收入确认**：先由团队确认，再由公司层面最终确认

### Case 1：新项目完整立项流程

**角色**：项目经理（提交）→ 团队经理（审核）

**触发场景**：公司拿到一个新的银行软件开发合同，需要正式立项管理。

#### 操作步骤

**步骤 1：项目经理提交立项申请**

1. 进入"项目管理 → 立项申请"页面
2. 填写项目基本信息：
   - 行业：选择"金融"
   - 一级区域：选择"华北"
   - 二级区域：自动关联显示华北省份，选择"北京市"
   - 简称：输入"ABC"
   - 立项年度：选择"2026"
   - 系统自动生成项目编号：`金融-HBQY-BJ-ABC-2026`
   - 项目名称：输入"ABC银行网银系统改造项目"
   - 项目分类：选择"软件开发类"
   - 项目部门：选择三级及以下部门，如"软件部门-创新组-项目五组"
3. 填写人员配置：项目经理、市场经理、参与人员、销售负责人
4. 填写客户信息：选择客户简称后联系人自动带出
5. 填写时间规划、成本预算
6. 点击"保存"
   - 系统自动：项目经理/市场经理/销售负责人/参与人员写入 `pm_project_member`
   - 系统自动：收入确认年度初始化为 `dd`（待定），状态初始化为 `0`（未确认）
   - 初始审核状态：`待审核`

**步骤 2：团队经理审核**

1. 进入"项目管理 → 立项审核"，看到待审核列表
2. 点击"审核"按钮，打开抽屉页面，查看项目全部信息
3. 操作：
   - **通过**：点击通过 → `approval_status = 1`，写入 `pm_project_approval` 记录
   - **拒绝**：填写审核意见 → `approval_status = 2`，写入拒绝记录
   - **退回**：→ `approval_status = 3`，项目经理可重新修改后提交

**结果**：项目在"项目管理"列表可见，状态为"审核通过"，可进行后续操作。

### Case 2：合同管理与款项跟踪

**角色**：经营管理人员

**触发场景**：项目立项通过后，签订了合同，需要录入合同信息并跟踪回款。

#### 操作步骤

**步骤 1：从项目关联合同**

1. 在项目管理列表，找到该项目，操作列显示"添加合同"（无合同时）
2. 点击"添加合同"，进入合同创建页面（预填了项目 ID、部门、客户信息）
3. 填写合同信息：
   - 合同名称、合同编号
   - 合同类型：选择"软件开发"
   - 合同金额：100万元（含税）
   - 税率：6%（软件开发类）
   - 系统自动计算：不含税金额 = 100 ÷ (1+6%) = 94.34 万元
   - 合同签订日期、合同周期
4. 保存合同，同时在 `pm_project_contract_rel` 写入项目-合同关联

**步骤 2：录入款项（付款里程碑）**

1. 进入"合同管理 → 款项管理"，新增款项
2. 填写付款方式名称，如"三期付款"
3. 填写里程碑明细：
   - 里程碑1：需求确认，30万
   - 里程碑2：上线验收，50万
   - 里程碑3：免维期满，20万
4. 设置付款状态：初始为"未开未付"
5. 设置预计回款季度：Q2/2026

**步骤 3：跟踪回款进度**

付款状态流转：`未开未付` → `已提交验收材料` → `验收材料已审核` → `待通知开票` → `已通知`

1. 验收完成后，编辑款项，更新付款状态
2. 实际回款时，填写"实际回款日期"和"实际回款季度"
3. 上传回款凭证附件（发票等）

**结果**：项目详情页可查看合同金额、各款项状态；款项管理页可按季度、状态汇总分析。

### Case 3：项目任务分解与排期

**角色**：项目经理

**触发场景**：项目批量上线前，需要将项目分解为多个子任务并安排投产批次。

#### 操作步骤

**步骤 1：创建投产批次**

1. 进入"任务管理 → 投产批次管理"
2. 新增批次：
   - 批次号：2026-B01
   - 投产年份：2026
   - 计划投产日期：2026-06-30
   - 排序：1

**步骤 2：分解项目任务**

1. 进入"任务管理 → 项目分解任务"
2. 选择机构：软件部门-创新组-项目五组
3. 选择项目：ABC银行网银系统改造项目（自动回显项目基本信息）
4. 查看本项目已分解的任务列表（当前为空）
5. 新增任务一：
   - 任务名称：网银前端界面开发
   - 任务编号：系统生成（父项目编号 + 0000001）
   - 投产年度：2026，投产批次：2026-B01（先选年度，再选批次）
   - 任务负责人：张工
   - 预估工作量：15人天
   - 软件中心需求编号：REQ-2026-001
   - 排期状态：已排期
6. 新增任务二：后端接口开发，关联同一批次

**步骤 3：查看任务汇总**

任务管理列表展示项目合并单元格：

```
项目名称           | 任务编号  | 任务名称         | 批次    | 排期状态 | 计划投产日期
ABC银行网银系统改造 | P001-0000001 | 网银前端界面开发 | 2026-B01 | 已排期 | 2026-06-30
（合并单元格）     | P001-0000002 | 后端接口开发     | 2026-B01 | 排期中 | 2026-06-30
```

### Case 4：员工日报填写与工时汇总

**角色**：员工

**触发场景**：员工每天下班前填写当日工作日报。

#### 操作步骤

**步骤 1：填写日报（普通项目）**

1. 进入"日报管理 → 日报填写"
2. 左侧月历显示当月每天的工时颜色（绿=≥8h，橙=<8h，灰=未填）
3. 点击今天的日期格，右侧切换到今天的编辑区
4. 看到参与的项目列表（仅显示状态"开启"且阶段未到"结项"的项目）
5. 对于"ABC银行网银系统改造"项目（无子任务），填写：
   - 工时：拖动滑块选择 6 小时
   - 工作任务类别（必选，多选）：选择"需求分析"
   - 工作内容：填写今日工作内容
6. 添加假期（如有）：点击上方"添加假期"，选择请假类型

**步骤 2：填写日报（有任务的项目）**

1. 对于有子任务的项目，展示展开的任务行
2. 每个任务行分别填写：
   - 工时（每个任务独立）
   - 工作任务类别（必选，多选）
   - 工作内容
3. 项目总工时 = 各任务工时之和（不可手动修改）

**步骤 3：保存日报**

点击"保存"，系统：
1. 写入 `pm_daily_report`（主表，一人一天唯一）
2. 写入 `pm_daily_report_detail`（各项目/任务明细）
3. 重算有任务的项目：各任务 `actual_workload` 累加
4. 汇总到父项目 `pm_project.actual_workload`
5. 汇总到主表 `pm_daily_report.total_work_hours`

**人天展示**：项目管理列表中该项目的"实际人天" = ROUND(actual_workload / 8, 3) + adjust_workload

### Case 5：工作日报动态（管理视角）

**角色**：团队经理/项目经理

**触发场景**：管理者需要查看团队成员的日报情况，了解工时分布。

#### 操作步骤

1. 进入"日报管理 → 工作日报动态"
2. 查询条件：
   - 选择姓名：张工
   - 选择日期范围：2026-03-01 ~ 2026-03-31
3. 页面以日历卡片形式展示 3 月份，每天格子内显示张工的日报摘要
   - 格子背景色：绿色（≥8h），橙色（<8h），灰色（未填）
4. 点击任意一天，弹出抽屉详情页：
   - 项目信息行：收入确认年度（非当年标红）、项目经理、项目名称（超链接）、项目阶段、预计人天、已花人天、工作任务类别
   - 任务信息行（如有）：任务编号、任务负责人、任务名称、任务阶段、预计人天、实际人天、工作任务类别

**项目经理权限特殊说明**：项目经理可查看所有参与该项目的人员日报，即使该人员不在自己直属团队。

### Case 6：公司收入确认

**角色**：团队负责人 → 公司财务

**触发场景**：季度末，需要对当季完成的项目进行收入确认。

#### 操作步骤

**步骤 1：团队收入确认**

1. 进入"收入确认管理 → 团队收入确认"
2. 查询部门下的项目，找到已完成的项目
3. 为该项目填写团队确认金额，保存写入 `pm_team_revenue_confirmation`

**步骤 2：公司收入确认**

1. 进入"收入确认管理 → 公司收入确认"
2. 列表顶部显示金额合计行
3. 找到该项目，点击"收入确认"按钮
4. 进入左右布局页面：
   - 左侧：项目详情信息（只读，不含备注和5个备用字段）
   - 右侧：填写确认信息
5. 填写确认信息：
   - 收入确认状态：选择"已确认"（字典：`sys_qrzt`）
   - 收入确认年度：选择"2026"
   - 确认日期：2026-03-31
   - 确认金额（含税）：94.34万元
   - 税率：6%（软件开发类）
   - 税后金额（系统自动计算）：94.34 ÷ (1+6%) = 89 万元
6. 点击保存，更新 `pm_project` 的确认相关字段

**结果**：项目管理列表中该项目的收入确认状态显示"已确认"，确认金额和税后金额可见。

### Case 7：项目经理变更

**角色**：团队经理/部门经理

**触发场景**：项目经理离职或调岗，需要批量变更多个项目的负责人。

#### 操作步骤

1. 进入"项目管理 → 项目经理变更"
2. 搜索：项目经理选择"张工"，查询张工负责的所有项目
3. 勾选需要变更的项目（支持全选）
4. 点击"批量修改"，弹出操作区，选择新项目经理"李工"
5. 确认操作
6. 系统：
   - 更新 `pm_project.project_manager_id`
   - 写入变更记录到 `pm_project_manager_change`（old_manager_id, new_manager_id）
   - 更新 `pm_project_member` 中的成员记录

**结果**：变更记录可追溯，每次变更保留完整记录（原经理、新经理、变更时间）。

### Case 8：附件管理全流程

**角色**：项目相关人员

**触发场景**：需要为项目/合同/款项上传相关文档。

#### 操作步骤

**上传附件**：

1. 在项目管理列表，点击"附件"按钮
2. 进入附件管理页
3. 选择文档类型（字典：`sys_wdlx`，如"立项文档"）
4. 点击"上传文件"，选择文件（≤30MB，支持 doc/pdf/xlsx 等）
5. 系统保存：
   - 写入 `pm_attachment`（记录元信息）
   - 写入 `pm_attachment_log`（操作类型=upload）
   - 文件存储到：`/app/uploadPath/project/{ID}_{项目名}/{yyyyMMdd}/{UUID}.pdf`

**查看日志**：

1. 在附件列表页，点击"日志"按钮
2. 查看该附件的所有操作记录：谁在什么时间进行了上传/下载/删除操作

**下载附件**：

1. 点击附件列表中的"下载"按钮
2. 系统写入 `pm_attachment_log`（操作类型=download）
3. 文件通过流下载到本地

### Case 9：团队日报（管理驾驶舱视角）

**角色**：团队经理

**触发场景**：月度复盘，需要查看团队内所有人员对各项目的投入情况，并分析是否超出预算人天。

#### 操作步骤

1. 进入"日报管理 → 团队日报"
2. 查询条件：
   - 项目部门：选择"项目五组"（自动包含下属子部门）
   - 日期：2026年3月
   - 收入确认年度：2026
3. 页面按日历卡片展示，每个格子内显示人员对项目的工时投入
4. **颜色维度**：是否带来收入的项目（有关联合同则高亮显示）
5. **分析维度**：
   - 实际人天 = 项目表 actual_workload/8 + adjust_workload
   - 预算人天 = estimated_workload
   - 对比提示：实际人天尽量不超过预算人天的一半

**关键指标**：当实际人天已超预算人天一半时，系统高亮提醒，团队经理需要介入管控。

### Case 10：人天补正操作

**角色**：项目管理员/系统管理员

**触发场景**：某项目的日报数据存在错误（漏填或多填），需要对实际人天进行手动补正。

#### 操作步骤

1. 在项目管理列表，找到需要补正的项目
2. 点击"收入确认"或进入项目统计页面
3. 触发人天补正操作，填写：
   - 调整方向：增加 or 减少
   - 调整人天数（delta）
   - 补正理由
4. 系统：
   - 记录 before_adjust（补正前值）
   - 更新 `pm_project.adjust_workload`
   - 写入 `pm_workload_correct_log`（审计日志，不可删除）
5. 项目列表"实际人天"列自动更新：
   - `ROUND(actual_workload / 8, 3) + adjust_workload`

**注意**：补正操作有完整审计日志，可追溯每次补正的时间、人员、理由。

### Case 11：工作日历维护（年度节假日设置）

**角色**：系统管理员

**触发场景**：每年年初，需要维护当年的法定节假日和调休工作日，以便日报模块正确判断哪些天需要填写日报。

#### 操作步骤

**步骤 1：批量设置年度日历**

1. 进入"工作日历管理"页面
2. 选择年份：2026
3. 录入所有特殊日期（非标准周末的变动）：
   - 元旦：2026-01-01，类型=holiday，名称="元旦"
   - 春节：2026-01-26 ~ 2026-02-01，类型=holiday，名称="春节"
   - 春节调休：2026-01-24（周六上班），类型=workday，名称="春节调休"
   - 清明、五一、端午、中秋、国庆等依此类推
4. 点击"批量保存"（`POST /project/workCalendar/batchSave`）
   - 系统先删除该年所有记录，再批量插入新记录

**步骤 2：日报模块自动引用**

1. 日报填写页的 MonthCalendar 组件通过 `GET /project/workCalendar/year/2026` 获取全年日历
2. 标记为 `holiday` 的日期，在日历上显示为灰色（无需填报）
3. 标记为 `workday` 的周末，在日历上显示为正常工作日（需要填报）

**结果**：日报填报统计能正确区分"应填未填"与"无需填写"，避免节假日被误统计为缺报。

### Case 12：日报白名单管理

**角色**：系统管理员 / 团队经理

**触发场景**：部分人员（高管、长期外派等）无需每日填写日报，需要将其加入白名单以免被"未提交"统计误报。

#### 操作步骤

**步骤 1：添加白名单**

1. 进入"日报管理 → 日报白名单"页面
2. 点击"添加"按钮
3. 选择人员（UserSelect 组件）
4. 填写加入原因：如"长期出差，不参与日常项目"
5. 保存，系统写入 `pm_daily_report_whitelist`

**步骤 2：白名单生效**

1. 日报填报统计（周统计）页面自动排除白名单人员
2. "未提交"人数不再包含白名单用户
3. 白名单用户本人可通过 `GET /project/whitelist/checkSelf` 确认自己的白名单状态

**步骤 3：移除白名单**

1. 在白名单列表页，找到需要移除的人员
2. 点击"移除"，执行软删除
3. 该人员重新纳入日报填报统计

**结果**：统计数据更加准确，不会因为特殊岗位人员未填日报而产生大量误报。

### Case 13：日报填报统计（周统计视角）

**角色**：团队经理 / HR

**触发场景**：每周需要检查团队成员日报填写情况，督促未填写人员补填。

#### 操作步骤

**步骤 1：查看周统计总览**

1. 进入"日报管理 → 日报填报统计"
2. 选择部门：项目五组（`GET /project/dailyReport/weeklyStatsDeptTree` 加载部门树）
3. 选择自然周（系统自动计算周一到周日的 startDate/endDate）
4. 页面展示表格：

```
姓名   | 部门     | 周一 | 周二 | 周三 | 周四 | 周五 | 周六 | 周日 | 合计
张工   | 项目五组 |  8h  |  8h  |  8h  |  8h  |  8h  |  —   |  —   | 40h  ✅
李工   | 项目五组 |  8h  |  6h  |  —   |  8h  |  8h  |  —   |  —   | 30h  ⚠️
王工   | 项目五组 |  —   |  —   |  —   |  —   |  —   |  —   |  —   |  0h  ❌
```

- 绿色：≥8h | 橙色：<8h | 灰色：未填

**步骤 2：查看明细**

1. 点击"已提交: 2人"链接 → `GET /project/dailyReport/weeklyStatsDetail?type=submitted`
2. 展示已提交人员清单及每日工时
3. 点击"未提交: 1人"链接 → `type=unsubmitted`
4. 展示未提交人员清单（排除白名单用户）

**步骤 3：导出报表**

1. 点击"导出"按钮 → `GET /project/dailyReport/weeklyStatsExport`
2. 下载包含当周所有人员填报情况的 Excel 文件

**结果**：管理者可快速定位未填报人员，支持导出作为考勤依据。

### Case 14：批量请假

**角色**：员工

**触发场景**：员工请假多天（如年假5天），需要一次性为多个日期填写假期记录，避免逐日手动添加。

#### 操作步骤

1. 进入"日报管理 → 日报填写"
2. 点击"批量请假"按钮
3. 填写：
   - 假期类型：年假（字典 `sys_gzlb` 中的假期子项）
   - 开始日期：2026-03-24
   - 结束日期：2026-03-28（共5个工作日）
   - 每天请假时长：8小时
4. 点击确认 → `POST /project/dailyReport/batchLeave`
5. 系统自动跳过该区间内的非工作日（参考工作日历），为每个工作日生成一条假期日报记录

**结果**：5天假期一次操作完成，月历上对应日期显示假期标记。

## 三、业务规则汇总

### 3.1 项目编号生成规则

```
格式：{行业代码}-{一级区域代码}-{二级区域代码}-{简称}-{立项年度}
示例：金融-HBQY-BJ-ABC-2026
字段最大长度：varchar(500)（已扩展，支持长简称）
唯一约束：uk_project_code
```

### 3.2 税务计算规则

```
软件开发类：税率 6%
硬件销售类：税率 13%
不含税金额 = 合同金额 ÷ (1 + 税率/100)
税金 = 不含税金额 × 税率
```

### 3.3 日报编辑限制

```
规则：仅当前自然周（周一至周日）可编辑
过去/未来周：只读
项目显示条件：project_status = '开启' AND project_stage ≠ '结项'
任务显示条件：task_stage ≠ '结项'
```

### 3.4 收入确认年度颜色规则

```
收入确认年度 < 当前年份 → 显示红色背景（提示需尽快确认）
其他（当年、待定、无法确认）→ 正常样式
```

### 3.5 部门树过滤规则

```
显示范围：三级及以下部门（一二级不出现）
根节点：三级部门
数据查询：选中任意节点 → 自动包含本级 + 所有子级
SQL 模式：find_in_set(#{deptId}, ancestors) > 0
```

### 3.6 项目-任务-日报 工时关系

```
日报明细（每条记录：project_id/sub_project_id + work_hours）
    ↓ 保存时自动汇总
任务实际工时（pm_task.actual_workload，存小时）
    ↓ 汇总
项目实际工时（pm_project.actual_workload，存小时）
    ↓ 展示时换算
实际人天 = ROUND(actual_workload / 8, 3) + adjust_workload
```

## 五、异常场景处理

| 场景 | 处理方式 |
|------|----------|
| 项目审核被拒绝 | 项目经理可查看拒绝原因，修改后重新提交（退回待审核状态） |
| 日报日期不在当前周 | 前端禁止编辑，显示只读状态 |
| 日报中工作任务类别未选择 | 保存时前端校验，必选项提示 |
| 合同编号重复 | 数据库唯一约束 + 后端校验 |
| 上传文件超过 30MB | 前后端双重校验，超出提示错误 |
| 附件类型不允许 | 前后端双重校验，不合法扩展名拒绝上传 |
| 项目人天异常（补正） | 通过 `pm_workload_correct_log` 记录调整过程，可追溯 |
| 项目阶段跳跃 | 系统拒绝，状态机约束，变更记录写入 `pm_project_stage_change` |
| PM 用户访问 system/ 接口 | 403 Forbidden，引导使用代理接口 |
| 重复提交 | `@RepeatSubmit` 注解防重，返回"请勿重复提交" |
| 白名单用户重复添加 | 后端校验 userId 唯一性，已存在则提示 |
| 批量请假日期含非工作日 | 系统自动跳过节假日/周末（参考工作日历），仅为工作日生成记录 |
