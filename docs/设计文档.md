### 2.3 Controller 规范

所有 Controller 继承 `BaseController`，list 方法首行调用 `startPage()`：

```java
@RestController
@RequestMapping("/project/project")
public class ProjectController extends BaseController {

    @PreAuthorize("@ss.hasPermi('project:project:list')")
    @GetMapping("/list")
    public TableDataInfo list(Project project) {
        startPage();   // 必须是第一行
        return getDataTable(projectService.selectProjectList(project));
    }

    @Log(title = "项目管理", businessType = BusinessType.INSERT)
    @PreAuthorize("@ss.hasPermi('project:project:add')")
    @PostMapping
    public AjaxResult add(@Validated @RequestBody Project project) {
        return toAjax(projectService.insertProject(project));
    }
}
```

**响应格式**：
- 列表：`TableDataInfo` → `{ code, msg, total, rows }`
- 单对象：`AjaxResult` → `{ code, msg, data }`
- 状态码：200=成功，401=重新登录，500=错误

### 2.4 自定义注解

| 注解 | 位置 | 作用 |
|------|------|------|
| `@Log(title, businessType)` | Controller 方法 | 异步写操作审计日志到 `sys_oper_log` |
| `@DataScope(deptAlias, userAlias)` | Service 方法 | 在 SQL 中注入 `${params.dataScope}` 数据权限过滤 |
| `@DataSource` | Service/方法 | 动态切换数据源 |
| `@RateLimiter(time, count)` | Controller 方法 | Redis Lua 限流 |
| `@RepeatSubmit` | Controller 方法 | 防重复提交 |
| `@Excel(name)` | 实体字段 | Excel 导出列配置 |
| `@Anonymous` | Controller 方法 | 跳过 JWT 认证（慎用） |
| `@PreAuthorize` | Controller 方法 | Spring Security 权限控制 |

### 2.5 异常处理

业务异常统一抛出 `ServiceException`，由 `GlobalExceptionHandler` 捕获并返回 `AjaxResult.error`：

```java
// 业务代码中
if (condition) {
    throw new ServiceException("操作失败：原因描述");
}

// GlobalExceptionHandler 自动处理，返回：
// { "code": 500, "msg": "操作失败：原因描述" }
```

### 2.6 数据权限设计

基于 `@DataScope` 注解，5种数据权限：

| 数据权限类型 | 说明 |
|-------------|------|
| 1 - 全部数据 | 无过滤条件 |
| 2 - 自定数据 | 按角色关联的指定部门过滤 |
| 3 - 本部门数据 | `dept_id = 当前用户部门` |
| 4 - 本部门及以下 | 使用 `ancestors` 层级查询 |
| 5 - 仅本人数据 | `create_by = 当前用户` |

部门层级查询（`find_in_set` 模式）：

```xml
<if test="projectDept != null and projectDept != ''">
    and (p.project_dept = #{projectDept}
         or p.project_dept in (
             select dept_id from sys_dept
             where find_in_set(#{projectDept}, ancestors) > 0
         ))
</if>
```

### 2.7 安全架构

```
请求 → JwtAuthenticationTokenFilter

         ↓ 加载 LoginUser（含权限列表）
         ↓ 设置 SecurityContextHolder
       ↓

         ↓
     @DataScope 数据权限 SQL 注入

```

### 3.5 搜索状态缓存（导航保持）

用于从列表页跳详情/编辑页返回时保持查询条件：

```typescript
const SEARCH_STATE_KEY = 'xxx_search_state'  // 每个列表页唯一

// 路由离开时保存
onBeforeRouteLeave(() => {
  sessionStorage.setItem(SEARCH_STATE_KEY, JSON.stringify({
    queryParams: { ...queryParams },
    asyncOptions: options.value  // 异步加载的下拉选项也要缓存
  }))
})

// 挂载时恢复
onMounted(() => {
  const raw = sessionStorage.getItem(SEARCH_STATE_KEY)
  if (raw) {
    const state = JSON.parse(raw)
    Object.assign(queryParams, state.queryParams)
    options.value = state.asyncOptions || []
    sessionStorage.removeItem(SEARCH_STATE_KEY)  // 一次性消费
  }
  getList()
})

// 手动重置时清除
function resetQuery() {
  sessionStorage.removeItem(SEARCH_STATE_KEY)
  // ...
}
```

## 四、数据库设计要点

### 4.1 命名规范

| 类型 | 规则 | 示例 |
|------|------|------|
| PM 业务表 | `pm_` 前缀 | `pm_project`, `pm_contract` |
| 系统表 | `sys_` 前缀 | `sys_user`, `sys_dept` |
| 代码生成表 | `gen_` 前缀 | `gen_table` |
| Quartz 表 | `qrtz_` 前缀 | `qrtz_triggers` |

### 4.2 字符集规范

| 表类型 | 字符集/排序规则 |
|--------|----------------|
| PM 业务表（大多数）| `utf8mb4` / `utf8mb4_0900_ai_ci` |
| 部分 PM 表（`pm_secondary_region`、`pm_project_manager_change`、`pm_work_calendar`、`pm_task` 等）| `utf8mb4` / `utf8mb4_unicode_ci` |
| RuoYi 系统表 | `utf8mb4` / `utf8mb4_unicode_ci` |

**跨字符集 JOIN 必须加 COLLATE**：

```sql
LEFT JOIN sys_user u
  ON p.update_by COLLATE utf8mb4_unicode_ci = u.user_name
```

### 4.3 审计字段（BaseEntity）

所有业务表均包含以下审计字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `create_by` | varchar(64) | 创建者用户名 |
| `create_time` | datetime | 创建时间 |
| `update_by` | varchar(64) | 更新者用户名 |
| `update_time` | datetime | 更新时间 |

**特别注意**：`pm_project.update_by / update_time` **只能**由"项目编辑/立项申请"功能更新，`syncProjectMembers()`、审批流、成员管理等操作禁止修改这两个字段。

### 4.4 软删除与硬删除

软删除使用 `del_flag` 字段（`'0'` = 正常，`'1'` = 删除）。

以下表使用**硬删除**，不使用 `del_flag`：

- `pm_project` — DELETE FROM pm_project
- `pm_task` — DELETE FROM pm_task（无 del_flag 列）
- `pm_production_batch` — DELETE FROM pm_production_batch（无 del_flag 列）
- `pm_daily_report` + `pm_daily_report_detail` — 事务中同时硬删除

### 4.5 人天计算公式

```sql
ROUND(p.actual_workload / 8, 3) + COALESCE(p.adjust_workload, 0) AS actual_workload_days
```

- `actual_workload` 字段存储的是**小时数**（由日报汇总而来）
- `adjust_workload` 存储的是**人天调整量**（人天补正功能）
- 禁止直接把 `actual_workload` 小时数作为人天展示

### 4.6 Schema 变更策略

| 场景 | 操作 |
|------|------|
| 新表 | 修改 `pm-sql/init/00_tables_ddl.sql` |
| 已部署 DB 的变更 | 创建 `pm-sql/fix_<feature>_<date>.sql`（已加入 `.gitignore`，不提交） |

## 五、业务流程设计

### 5.1 项目状态机

```
立项申请（待审核）
      ↓ 提交
   待审核(0)
    ↓      ↓
 通过(1)  拒绝(2)
    ↓
 (可退回 → 退回待审核(3) → 重新提交 → 待审核(0))
```

审批操作仅通过 `ProjectApprovalController` 专用接口进行，业务代码禁止直接修改 `pm_project.approval_status`。

### 5.2 收入确认流程

```
未确认(0) → 待确认 → 已确认 / 无法确认
              ↑
         字典：sys_qrzt
```

公司收入确认金额计算：
- 税后金额 = 确认金额 ÷ (1 + 税率/100)
- 例：100 ÷ (1 + 6%) = 94.34

### 5.3 日报工时汇总链

```
pm_daily_report_detail.work_hours（每条日报明细）
          ↓ 保存时汇总
pm_task.actual_workload（子任务实际工时）
          ↓ 汇总
pm_project.actual_workload（父项目实际工时）
          ↓ 计算展示
ROUND(actual_workload / 8, 3) + adjust_workload （人天）
```

### 5.4 项目成员同步

立项申请/保存时，调用 `syncProjectMembers()`：

- 将 `projectManagerId`、`marketManagerId`、`salesManagerId`、`teamLeaderId` 和 `participants` 全部同步写入 `pm_project_member`
- 子任务成员**不**单独写入，继承父项目成员列表
- `syncProjectMembers()` 操作不更新 `pm_project` 的 update 时间戳

### 5.5 附件存储路径

```
/app/uploadPath/
└── {业务类型}/             (project | contract | payment)
    └── {ID}_{名称}/
        └── {yyyyMMdd}/
            └── {UUID}.{ext}
```

K3s 生产环境挂载 `upload-pvc` 到 `/app/uploadPath`。

## 九、关键设计决策记录

| 决策 | 原因 |
|------|------|
| 任务独立拆分为 `pm_task` 表 | 原项目表任务字段越来越多，关注点分离，减少技术债 |
| 日报明细 `sub_project_id` 保留原列名 | 向后兼容，实际存储的是 `pm_task.task_id` |
| PM 表使用代理接口而非直接调用 system/ | PM 角色无 system 权限，且避免绕过数据权限隔离 |
| `pm_project` 和 `pm_task` 使用硬删除 | 历史数据重要，软删除会引入脏数据复杂性 |
| 日报使用事务硬删除 | 日报主表+明细必须同步删除，避免孤儿记录 |
| 字符集跨表 JOIN 加 COLLATE | `utf8mb4_0900_ai_ci` 与 `utf8mb4_unicode_ci` 排序规则不同，直接 JOIN 会报错 |
| 搜索状态用 sessionStorage | tab 级别隔离，不跨 Tab 污染，刷新后自动清空 |
| 宪法优先于局部惯例 | `.specify/memory/constitution.md` 定义了 6 条不可违背原则 |
