# Feature Specification: 日报填写右侧查询条件

**Feature Branch**: `013-daily-report-write-filter`
**Created**: 2026-06-26
**Status**: Draft
**Input**: User description: "日报管理-日报填写 在右半部分的上方，添加查询条件：项目名称、任务名称的模糊查询、项目经理的下拉查询。因为有的人关联的项目或者任务太多 不方便查找。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 按项目名称模糊查找项目 (Priority: P1)

填日报的员工往往同时挂在很多项目下。打开「日报填写」页面右侧编辑区时，会一次性铺开所有参与项目，列表很长、滚动很久才能找到要填的那一个。员工希望在右侧编辑区顶部输入一段项目名称关键字，列表立即只剩下名称包含该关键字的项目，从而快速定位。

**Why this priority**: 这是用户痛点的核心——"关联的项目太多不方便查找"。仅实现项目名称模糊过滤，就已经能显著缩短定位时间，构成可独立交付的最小可用增量。

**Independent Test**: 用一个参与多个项目的账号登录，进入日报填写页，在右侧顶部项目名称输入框键入某项目名片段，验证列表实时只显示名称匹配的项目；清空输入后列表恢复完整。

**Acceptance Scenarios**:

1. **Given** 当前用户参与 8 个项目且右侧编辑区已全部展示，**When** 在项目名称输入框输入某项目名称的一个片段，**Then** 右侧列表实时只显示项目名称包含该片段的项目卡片（不区分大小写）。
2. **Given** 项目名称输入框已有过滤关键字，**When** 清空该输入框，**Then** 右侧列表恢复显示全部参与项目。
3. **Given** 用户在某项目卡片已填写了工时，**When** 输入项目名称关键字使该项目卡片被过滤隐藏，再清空关键字使其重新显示，**Then** 该项目卡片中已填写的工时与工作内容保持不变（过滤不丢数据）。

---

### User Story 2 - 按任务名称模糊查找任务 (Priority: P2)

有子任务的项目会展开成多行任务，任务数量也可能很多。员工希望在右侧顶部输入任务名称关键字，快速定位到含有匹配任务的项目，并只看到匹配的任务行，避免在大量任务行中翻找。

**Why this priority**: 任务级查找是项目级查找的补充，覆盖"关联的任务太多"的场景。它依赖项目卡片渲染逻辑，价值次于项目名称查找，故为 P2。

**Independent Test**: 用一个参与含多子任务项目的账号，在任务名称输入框键入某任务名片段，验证只显示含匹配任务的项目卡片，且卡片内只展示名称匹配的任务行。

**Acceptance Scenarios**:

1. **Given** 当前用户参与的项目中有若干含子任务的项目，**When** 在任务名称输入框输入某任务名片段，**Then** 右侧只显示"至少含一个名称匹配任务"的项目卡片，且这些卡片内只展示名称匹配的任务行（不区分大小写）。
2. **Given** 任务名称输入框有关键字，**When** 某项目不含任何子任务（普通项目），**Then** 该普通项目卡片不出现在结果中（任务名称查找只针对任务）。
3. **Given** 任务名称过滤后某些任务行被隐藏，**When** 用户保存日报，**Then** 被隐藏任务行上已录入的工时仍会被正常保存（过滤只影响显示，不影响保存数据）。

---

### User Story 3 - 按项目经理下拉筛选 (Priority: P3)

员工希望按"项目经理"维度收窄列表——例如只想看某位项目经理负责的项目，再在其中找要填的项目/任务。下拉选项来自当前用户参与项目的项目经理集合。

**Why this priority**: 项目经理筛选是又一个收窄维度，适合配合前两个模糊查询联合使用，属于锦上添花，故为 P3。

**Independent Test**: 用参与多个项目（分属不同项目经理）的账号，在项目经理下拉选择某位经理，验证只显示该经理负责的项目卡片。

**Acceptance Scenarios**:

1. **Given** 当前用户参与的项目分属多位项目经理，**When** 展开项目经理下拉，**Then** 下拉选项为这些项目去重后的项目经理集合（不含与当前用户无关的人员）。
2. **Given** 在项目经理下拉选择某位经理，**When** 过滤生效，**Then** 右侧只显示该项目经理负责的项目卡片。
3. **Given** 已选中某项目经理，**When** 清空下拉选择，**Then** 列表恢复（不再按经理过滤）。

---

### Edge Cases

- **三个条件同时设置**：项目名称、任务名称、项目经理三者以"且(AND)"关系组合——只有同时满足全部已填条件的项目（及其匹配任务行）才显示。
- **过滤后无结果**：当没有项目满足过滤条件时，右侧编辑区显示友好的空状态提示（如"没有符合筛选条件的项目"），而非空白。
- **过滤期间保存日报**：保存逻辑遍历的是完整的内部表单数据，而非当前可见列表；因此被过滤隐藏的项目/任务上已录入的工时不会丢失，仍会一并保存。
- **白名单用户**：被设为"无需填写日报"的用户，右侧编辑区本就不展示，查询条件栏也不展示。
- **暂无参与项目**：当前用户没有任何参与项目时，查询条件栏可不展示或展示但无可过滤项；保持原"暂无参与的项目"提示。
- **切换日期**：在不同日期间切换时，已设置的查询条件保留，无需每次重新输入。
- **假期记录区块**：查询条件只作用于项目/任务列表，不影响"假期记录"区块的展示。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 在「日报填写」页右侧编辑区的顶部（项目/任务列表上方）展示一个查询条件栏，包含：项目名称输入框、任务名称输入框、项目经理下拉框。
- **FR-002**: 系统 MUST 支持按项目名称对右侧项目列表进行模糊（子串、不区分大小写）实时过滤——仅显示项目名称包含输入关键字的项目卡片。
- **FR-003**: 系统 MUST 支持按任务名称对右侧列表进行模糊（子串、不区分大小写）实时过滤——仅显示含有名称匹配任务的项目卡片，并在这些卡片内仅展示名称匹配的任务行。
- **FR-004**: 系统 MUST 提供项目经理下拉过滤，下拉选项 MUST 来自当前用户参与项目去重后的项目经理集合；选择某经理后仅显示该经理负责的项目卡片。
- **FR-005**: 系统 MUST 在多个查询条件同时设置时，以"且(AND)"关系组合过滤。
- **FR-006**: 系统 MUST 在过滤为纯前端展示操作，不向后端发起新的列表请求，过滤结果实时随输入/选择变化。
- **FR-007**: 系统 MUST 保证过滤操作不修改、不丢弃已录入的工时/工作内容/工作类别数据——被隐藏的项目或任务行上已填的数据在保存时仍被持久化。
- **FR-008**: 系统 MUST 在任一查询条件被清空（输入框清空 / 下拉清空）时，相应地放宽过滤；当全部条件为空时，右侧列表恢复显示全部参与项目（及其全部任务行）。
- **FR-009**: 系统 MUST 在过滤后无任何项目满足条件时，于右侧编辑区显示空状态提示，而非空白区域。
- **FR-010**: 系统 MUST 在用户于不同日期间切换时保留已设置的查询条件（条件为会话内的视图状态，不随日期切换被重置）。
- **FR-011**: 查询条件栏 MUST 仅在右侧编辑区可见的场景下出现（白名单用户、无右侧编辑区时不展示）。

### Key Entities *(include if feature involves data)*

- **参与项目项 (Project item)**：当前用户参与的某个项目在右侧列表中的一行，关键属性含项目名称、项目经理、是否含子任务。是项目名称过滤与项目经理过滤的作用对象。
- **任务行 (Task row)**：含子任务项目下展开的某个任务，关键属性含任务名称。是任务名称过滤的作用对象。
- **查询条件 (Filter criteria)**：会话内的视图状态，含项目名称关键字、任务名称关键字、所选项目经理。不持久化到后端。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在参与 ≥10 个项目的账号上，用户可在 10 秒内通过输入项目名称关键字定位到目标项目（对比无过滤时需逐屏滚动查找）。
- **SC-002**: 输入/选择任一查询条件后，列表过滤结果在 1 秒内（无明显延迟）实时反映，无需点击额外的"查询"按钮。
- **SC-003**: 在任意过滤状态下完成保存，已录入工时 100% 被持久化，无因过滤导致的数据丢失。
- **SC-004**: 项目经理下拉选项 100% 来自当前用户的参与项目集合，不出现无关人员。
- **SC-005**: 清空全部查询条件后，列表 100% 恢复到过滤前的完整项目/任务展示。

## Assumptions

- 右侧编辑区的项目列表与任务行数据在页面加载时已一次性取得（来自当前用户的参与项目接口及任务选项接口），因此三项查询均为前端对已加载数据的过滤，无需新增或修改后端接口。
- "项目经理"指项目级别的项目经理（非任务负责人）。下拉以项目经理姓名为展示与过滤依据。
- 任务名称模糊查询只针对含子任务项目的任务行；不含子任务的普通项目在任务名称条件被设置时不计入匹配结果。
- 模糊查询为大小写不敏感的子串匹配（中文不涉及大小写，英文/编号片段大小写不敏感）。
- 过滤为"显示/隐藏"语义，不从内部表单数据结构中删除任何条目，从而保证保存逻辑遍历完整数据、不丢工时。
- 查询条件为会话内视图状态，刷新页面后可重置；不要求跨刷新持久化。
- 本特性为纯前端改动，作用于 `日报填写` 页面右侧编辑区，不改动左侧日历、假期记录、保存/删除等既有逻辑。
