# UI 过滤契约：日报填写右侧查询条件

**Feature**: 013-daily-report-write-filter

本特性无网络接口契约。此处定义**前端 UI 行为契约**：给定查询条件输入，右侧编辑区可见结果的确定性映射。e2e 测试据此断言。

## 元素契约（查询条件栏）

| 元素 | 类型 | 占位/标签 | 行为 |
|---|---|---|---|
| 项目名称输入 | `el-input` clearable | `placeholder="项目名称"` | 输入即时过滤；清空恢复 |
| 任务名称输入 | `el-input` clearable | `placeholder="任务名称"` | 输入即时过滤；清空恢复 |
| 项目经理下拉 | `el-select` clearable | `placeholder="项目经理"` | 选项=参与项目经理去重集合；选中即时过滤；清空恢复 |

- 位置：右侧编辑区（`el-col span=17`）卡片内，项目/任务列表（含假期记录区块）**上方**。
- 仅在 `!isWhitelisted` 且右侧编辑区可见时渲染。

## 输入→可见结果映射（契约用例）

设当前用户参与项目：P1「营销系统」(经理=张三, 普通)、P2「核心账务」(经理=李四, 含任务 T1「批量入库」/T2「报表导出」)、P3「风控平台」(经理=张三, 普通)。

| # | 项目名 | 任务名 | 经理 | 期望可见 |
|---|---|---|---|---|
| C1 | （空） | （空） | （空） | P1, P2, P3（全部；P2 含 T1,T2） |
| C2 | `账务` | （空） | （空） | 仅 P2 |
| C3 | （空） | `入库` | （空） | 仅 P2，且仅显示任务行 T1 |
| C4 | （空） | （空） | 张三 | P1, P3 |
| C5 | （空） | `导出` | 李四 | 仅 P2，且仅显示 T2（AND 组合） |
| C6 | `营销` | `入库` | （空） | 空结果（P1 无匹配任务）→ 显示空状态提示 |
| C7 | 清空 C2 的项目名 | （空） | （空） | 恢复 P1, P2, P3 |

## 不变量契约

- **保存不丢数据**：在任意过滤状态（如 C3 隐藏了 P2.T2）下点击保存，被隐藏任务/项目上已录入的工时仍被持久化。
- **大小写不敏感**：项目名 `账务` / 英文片段大小写不敏感子串匹配。
- **空状态**：可见结果为 0 且存在过滤条件 ⇒ 显示「没有符合筛选条件的项目」，而非空白。
