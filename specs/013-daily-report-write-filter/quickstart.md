# Quickstart：本地验证日报填写查询条件

**Feature**: 013-daily-report-write-filter

## 前置

- 后端运行于本地 **8085**（`application.yml` `server.port`），MySQL 3306 / Redis 6379 已起。
- 前端 `cd ruoyi-ui && npm run dev`（端口 80，`/dev-api` 代理 → 8085）。
- 账号 `admin` / 密码 `123456789`。
- 跑 e2e 前临时**关闭登录验证码**（用户记忆 `feedback_e2e_captcha_toggle`），跑完恢复。

## 手动验证步骤

1. 浏览器打开 `http://localhost/project/dailyReport/write`，登录。
2. 右侧编辑区顶部应出现一行查询条件：**项目名称**输入、**任务名称**输入、**项目经理**下拉。
3. 在「项目名称」输入某项目名片段 → 右侧只剩名称匹配的项目卡片；清空 → 恢复全部。
4. 在「任务名称」输入某任务名片段 → 只剩含匹配任务的项目，且卡片内只显示匹配任务行；普通（无子任务）项目消失。
5. 「项目经理」下拉 → 选项为你参与项目的经理去重集合；选一个 → 只剩该经理的项目；清空 → 恢复。
6. 三者组合（AND）：同时设项目名+经理 → 仅同时满足者可见。
7. 无匹配时 → 显示「没有符合筛选条件的项目」。
8. **关键**：在某项目填好工时，再用查询条件把它隐藏，然后点「保存日报」→ 重新打开当天，工时仍在（过滤不丢数据）。
9. 切换左侧日历到另一天 → 查询条件保留，不被清空。

## 自动化 e2e

```bash
# 确保前端(80)+后端(8085)在跑，且验证码已关
npx playwright test e2e-daily-report-write-filter.spec.js
npx playwright show-report   # 查看结果
```

测试覆盖：条件栏存在性、项目名过滤、任务名过滤（含任务行收窄）、经理下拉过滤、清空恢复、空状态、保存不丢工时（契约用例 C1–C7）。
