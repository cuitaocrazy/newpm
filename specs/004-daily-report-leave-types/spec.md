# Feature Specification: 日报假期模块扩展

**Feature Branch**: `004-daily-report-leave-types`
**Created**: 2026-03-20
**Status**: Draft
**Input**: 日报假期模块新增婚假、产假、丧假类型，支持批量填写假期（按日期范围一键填入），并将假期类型颜色改为字典驱动

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 批量填写长假假期 (Priority: P1)

员工请产假、婚假等连续多天假期时，无需逐天点击日历手动填写，而是通过「批量填假期」入口，选择假期类型和日期范围，系统自动为范围内的每个工作日生成对应的假期记录，节省大量重复操作。

**Why this priority**: 这是本次需求的核心驱动力——现有「每天单独填」的方式对产假这类长假极不友好，批量填写直接解决该痛点，且独立可交付。

**Independent Test**: 选择婚假 + 日期范围（含周末）点击提交，验证范围内各工作日是否正确创建了假期记录，且周末被跳过。

**Acceptance Scenarios**:

1. **Given** 员工在日报填写页，**When** 点击「批量填假期」，选择「婚假」+ 开始日 2026-03-23、结束日 2026-03-25，确认提交，**Then** 系统为 03-23（周一）、03-24（周二）、03-25（周三）各创建一条婚假 8h 记录，并在日历角标上展示。
2. **Given** 选择的日期范围内含周末（如 03-21 ~ 03-25），**When** 提交，**Then** 周末日期（03-21、03-22）不生成记录，仅工作日生成。
3. **Given** 某天已有假期记录，**When** 批量填写范围覆盖该天，**Then** 系统给出冲突提示，由用户选择跳过或覆盖。
4. **Given** 员工批量填写后查看某天日报，**When** 点击该日期，**Then** 假期记录行已显示且可单独编辑或删除。

---

### User Story 2 - 使用新假期类型记录单日假期 (Priority: P2)

员工某天请婚假/产假/丧假，在单日日报填写页的「假期记录」区块，可从类型下拉中直接选到这三种新类型，操作与现有请假/倒休/年假完全一致。

**Why this priority**: 是基础能力，批量填写也依赖新类型存在，但单日填写本身已能满足短假场景（如丧假 3 天逐日填写仍可接受）。

**Independent Test**: 在假期记录区块添加一行，选择「婚假」，设置小时数并保存，验证保存成功且日历角标显示「婚假」标识。

**Acceptance Scenarios**:

1. **Given** 员工打开本周任意日期的日报，**When** 在假期记录区块点击「+ 添加假期」，**Then** 类型下拉中出现「婚假」「产假」「丧假」三个选项。
2. **Given** 选择「产假」并填写 8h，**When** 保存，**Then** 日历该日显示产假角标，颜色区别于其他假期类型。
3. **Given** 查看统计或团队日报，**When** 展示该员工当天记录，**Then** 产假/婚假/丧假的 label 正确显示汉字名称而非 key 值。

---

### User Story 3 - 假期颜色与字典同步 (Priority: P3)

当系统字典中配置了假期类型的颜色，前端日历角标和假期区块的颜色圆点自动使用字典中的颜色，无需修改前端代码。

**Why this priority**: 属于可维护性改进，不影响当前业务流程，可独立上线，未来扩展假期类型时零代码改动。

**Independent Test**: 在字典 `sys_rbtype` 中新增一条测试类型并配置颜色，不改前端代码，验证日报页面该类型颜色圆点是否与字典一致。

**Acceptance Scenarios**:

1. **Given** 字典中某假期类型配置了颜色标识，**When** 在日报填写页使用该类型，**Then** 颜色圆点和日历角标颜色与字典配置一致。
2. **Given** 新增字典类型未在前端代码中硬编码，**When** 用户使用该类型，**Then** 系统使用字典颜色展示，而非默认灰色兜底。

---

### Edge Cases

- 批量填写日期范围内全为周末/节假日时，系统提示「所选范围内无工作日，无记录生成」。
- 批量填写跨月（如 2026-03-30 ~ 2026-04-05）时，系统正确处理跨月日期，并分别在对应月份日历上显示。
- 某天已存在工时记录（work 条目），批量填写假期不影响工时记录，两者并存。
- 员工在批量填写弹窗中选择开始日晚于结束日时，系统给出校验提示，阻止提交。
- 批量填写日期范围超过 180 天时，系统给出警告提示（防误操作），需用户二次确认。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 在日报条目类型字典中支持「婚假」（marriage）、「产假」（maternity）、「丧假」（bereavement）三种新类型。
- **FR-002**: 日报填写页的假期类型下拉 MUST 展示字典中所有非工时类型，包括新增的三种。
- **FR-003**: 系统 MUST 提供「批量填假期」功能入口，允许用户选择假期类型 + 日期范围后一键为多个工作日生成假期记录。
- **FR-004**: 批量填写时，系统 MUST 自动跳过日期范围内的周末；若工作日历中标记了节假日，同样跳过。
- **FR-005**: 批量填写时，若目标日期已存在假期记录，系统 MUST 给出冲突提示，允许用户选择跳过或覆盖。
- **FR-006**: 批量填写默认每天 8 小时，用户 MUST 能在批量填写弹窗中统一修改时长后再提交。
- **FR-007**: 前端日历角标和假期区块颜色圆点 MUST 由字典颜色配置驱动，不得硬编码颜色值。
- **FR-008**: 批量填写结果 MUST 在月历视图中立即反映（角标更新），无需手动刷新。
- **FR-009**: 批量填写日期范围 MUST 支持跨月选择。
- **FR-010**: 批量填写不受「仅限本周可填写」的限制，允许为历史和未来日期批量生成假期记录。

### Key Entities

- **假期记录条目**: 每天每种假期类型一条记录，包含假期类型（字典 key）、时长（小时）、备注。与工时条目并存于同一日报下。
- **假期类型字典**: 驱动前端展示的类型列表、名称、颜色；新增类型只需维护字典，无需改代码。
- **批量填写请求**: 包含假期类型、日期范围（开始/结束）、每日时长、冲突处理策略（跳过/覆盖）。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 员工填写 10 天产假的操作步骤从逐日填写的约 10 次独立操作缩短为批量填写的 3 步（选类型 → 选范围 → 确认），操作次数减少 70% 以上。
- **SC-002**: 新增的三种假期类型在下拉、日历角标、统计、团队日报等所有展示场景中 label 均正确显示中文名称。
- **SC-003**: 批量填写时系统 100% 准确识别并跳过周末，不产生误填的工作日。
- **SC-004**: 颜色配置完全由字典驱动后，新增任意假期类型无需前端代码改动即可在页面正确着色。

## Assumptions

- 批量填写放开「仅限本周」限制，允许为历史和未来日期批量填假期，以支持产假等事后/提前补录场景。
- 批量填写默认时长为 8 小时/天，对应全天假期；半天假期仍通过单日填写处理。
- 系统不做法定假期天数校验（如婚假 3 天上限），由 HR 流程保障，系统只做如实记录。
- 陪产假（男性）暂不作为独立类型，如需区分可后续通过字典扩展，本次不在范围内。
- 颜色驱动方案复用现有 Element Plus tag 标准色系（primary/success/warning/danger/info）映射颜色，不引入自定义 hex 色值。
